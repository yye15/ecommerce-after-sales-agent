# Agent 评估方案

本项目使用人工审核的电商售后评估集衡量系统效果。评估重点不是“回复是否像人”，而是售后运营里更关键的指标：能否识别高风险问题、能否减少风险低估、能否正确归因问题类别。

## 评估集

当前主要评估文件：

```text
data/eval/customer_service_eval_100.csv
```

规模：

```text
100 条中文电商评论 / 售后对话
```

每条样本包含：

- `review_text`：评论或售后对话文本
- `expected_categories`：人工期望问题分类
- `expected_risk_level`：人工期望风险等级
- `expected_priority`：人工期望优先级
- `expected_rule_reason`：人工判断依据

## 运行评估

离线规则模式：

```powershell
python scripts\evaluate_golden_set.py --input data\eval\customer_service_eval_100.csv --output data\eval\customer_service_eval_100_results.csv --errors-output data\eval\customer_service_eval_100_errors.csv --summary-output data\eval\customer_service_eval_100_summary.json
```

调用 DeepSeek 的模式：

```powershell
python scripts\evaluate_golden_set.py --use-llm
```

## 输出文件

```text
data/eval/customer_service_eval_100_results.csv
data/eval/customer_service_eval_100_errors.csv
data/eval/customer_service_eval_100_summary.json
```

其中：

- `customer_service_eval_100_results.csv`：逐条样本预测结果
- `customer_service_eval_100_errors.csv`：只保留错误或部分错误案例，方便迭代
- `customer_service_eval_100_summary.json`：整体指标摘要

## 指标解释

- **风险等级准确率**：系统预测的低 / 中 / 高风险是否与人工标注一致。
- **高风险召回率**：人工认为高风险的样本中，系统抓住了多少。这是售后系统最重要的指标之一。
- **高风险精确率**：系统判为高风险的样本中，确实是高风险的比例。
- **风险低估率**：人工认为更严重、系统却判轻的比例。售后场景中这个指标越低越好。
- **问题分类命中率**：系统预测的问题类别与人工类别是否有交集。
- **问题分类完全匹配率**：系统预测类别集合与人工类别集合是否完全一致。
- **优先级准确率**：系统输出的 P1/P2/P3 是否符合人工期望。

## 本轮规则迭代

本轮先处理 `risk_underestimated`，因为低估高风险是售后场景中最危险的问题；随后升级了问题分类体系，使分类结果从单一粗标签变成“一级类别 + 业务维度 + 二级原因”的结构。新增或调整了以下规则：

- 核心质量问题：屏幕裂开、机器异常、听筒故障、花屏、换机后仍异常等直接进入高风险或中高风险。
- 售后阻塞问题：退货、换货、检测、验货、客服不处理等与负面质量问题叠加时提高风险。
- 明确缓和信号：例如“不是京东的问题”“最后让验货”等会降低误判风险，但不会覆盖严重质量信号。
- 正向否定过滤：例如“没坏”“没漏”等不再误触发商品质量问题。
- 分类体系增加业务维度：商品、体验、履约、服务、权益、售后。
- 分类体系增加二级原因：核心故障、耐用性不足、食品新鲜度、退款诉求、响应慢或失联等。
- 对“屏幕显示有待提高”“房间设施达不到四星水平”等轻微体验抱怨保留分类，但不再自动升级为中风险。

## 剩余问题

当前风险判断已经在 100 条评估集上全部命中，剩余错误主要来自问题分类边界：

```text
category_partial: 35
category_miss: 6
```

这说明下一步不应继续硬调风险规则，而应该重新梳理分类标签边界，尤其是：

1. `商品质量问题` 和 `产品体验问题` 是否允许同时出现。
2. 酒店、图书、水果等非标准电商品类是否需要独立标签。
3. `其他反馈` 是否过宽，是否需要拆成更具体类别。
4. 是否需要引入二级分类，例如 `质量问题 -> 耐用性 / 破损 / 核心功能异常 / 假货质疑`。

## 下一步优化方向

建议按下面顺序继续：

1. 固化当前风险规则，避免继续对 100 条样本过拟合。
2. 扩大评估集到 150-200 条，验证风险指标是否稳定。
3. 对人工评估集同步标注二级原因，进一步衡量细粒度分类质量。
4. 增加面向项目展示的评估图表和 README 指标说明。
5. 再把 LALUN 中文实验结果作为细粒度情感分析模块接回主页面。
