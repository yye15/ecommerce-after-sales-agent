# 电商售后风险评分标准 v1

这个标准用于把客户评论、投诉或售后对话转换成可解释的风险等级。它不是单纯判断情绪好坏，而是判断商家应该如何优先处理。

## 风险等级

| 等级 | 优先级 | 含义 | 分数范围 |
|---|---|---|---|
| 高风险 | P1 | 可能引发投诉、维权、退款纠纷、安全问题或舆情扩散，需要人工优先介入 | 82-100 |
| 中风险 | P2 | 存在明确负面体验或核心业务问题，需要客服主动处理 | 55-81 |
| 低风险 | P3 | 普通反馈、轻微问题或正向评价，按标准流程处理 | 0-44 |

## 关键规则

| 规则 | 处理方式 |
|---|---|
| 安全、健康、食品安全、漏电、过敏、烫伤等问题 | 直接高风险 |
| 投诉、举报、维权、12315、起诉、曝光、公开差评威胁 | 直接高风险 |
| 假货、欺骗、被骗、空单号等信任问题 | 直接高风险 |
| 退款/退货/换货 + 客服不回复或售后不处理 | 直接高风险 |
| 物流履约问题 + 退款/权益诉求 | 直接高风险 |
| 商品质量、耐用性、核心功能负面 | 至少中风险 |
| 客服响应差但暂未出现退款/投诉 | 中风险 |
| 物流慢但语气温和、未出现退款/投诉 | 低风险或中风险，优先常规跟进 |
| 纯正面评价 | 低风险 |

## 输出字段

风险节点会输出：

- `level`：低风险 / 中风险 / 高风险
- `score`：0-100 的风险分数
- `priority`：P1 / P2 / P3
- `should_escalate`：是否建议人工升级
- `reasons`：通俗解释
- `triggered_rules`：命中的规则明细
- `rule_version`：当前规则版本
